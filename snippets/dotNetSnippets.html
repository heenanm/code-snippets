# Interfaces

# Class Interface

  • Made up of the public members and methods.
  • This is how the class looks to the rest of the application.

# Interfaces

  • Defines signature of properties and methods.
  • Does not include any implementation.

# Defining Interfaces

  • Add a new interface to the project.
  • Prefix the interface name with an “I”.
  • Specify the public access modifier.
  • Add the property, method, event or indexer required.
  • No need for access modifier on interface members they are all public. *No implementation of the members - no method code only signatures.

# Implementing Interfaces

  • Add the Interface to the class signature.
  • Implement each member of the interface.
using System; 

namespace MyInterfaceExample 
{ 

    public interface IMyLogInterface 
    { 
        //I want to have a specific method that I'll use in MyLogClass 
        void WriteLog(); 
    } 

    public class MyClass : IMyLogInterface 
    { 
        public void WriteLog() 
        {
            Console.Write("MyClass was Logged"); 
        }
    }

    public class MyOtherClass : IMyLogInterface 
    { 
        public void WriteLog() 
        { 
            Console.Write("MyOtherClass was Logged"); 
            Console.Write("And I Logged it different, than MyClass"); 
        } 
    } 

    public class MyLogClass 
    { 
        //I created a WriteLog method where I can pass as a parameter any object that implements IMyLogInterface. 
        public static void WriteLog(IMyLogInterface myLogObject) 
        { 
            myLogObject.WriteLog(); 
            //So I can use WriteLog here. 
        }    
    } 

    public class MyMainClass 
    { 
        public void DoSomething() 
        { 
            MyLogClass logClass = new MyLogClass();
            MyClass aClass = new MyClass(); 
            MyOtherClass otherClass = new MyOtherClass(); 
            logClass.WriteLog(aClass);
            //MyClass can log, and have his own implementation 
            logClass.WriteLog(otherClass); 
            //As MyOtherClass also have his own implementation on how to log. 
        } 
    }
}

# Swagger configuration

services.AddSwaggerGen(c =>
           {
               c.CustomSchemaIds(x => x.FullName);
               c.SwaggerDoc("v1", new OpenApiInfo { Title = "My API", Version = "v1" });
           });

Using the custom schema ids in the swagger gen options will fully qualify the namespace on the objects used by the controllers. This will stop errors due to naming conflicts if you use the same names for objects.

Eg. create client may use a client object that has a name and description, where find client may use a client object that has a client Id.

# API Controller attribute

[ApiController]
public class CreateClientController : Controller

When creating controller classes add the attribute shown so that the post request body is implemented. If the attribute is not used then the post request will be sent as a query string.

# API Task< IActionResult >

[HttpPost("/clients/create")]
        public async Task<IActionResult> CreateClientAsync(CreateClientRequest request)
        {
            var newClient = new Core.Entities.Client
            {
                ClientId = Guid.NewGuid(),
                ClientName = request.Name,
                Description = request.Description,
                ClientTags = request.ClientTags.Select(ct => new Core.Entities.ClientTag
                {
                    ClientTagId = Guid.NewGuid(),
                    Text = ct.Text
                }).ToList()
            };
 
            _context.Client.Add(newClient);
 
            await _context.SaveChangesAsync();
 
            return Ok();

Task < IActionResult > allows the return of status codes such as return Ok(); which will send a 200 response.
