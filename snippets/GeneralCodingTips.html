Encapsulation Well written classes are changable make all member variables private - code outside the class can't count on the names and types. You can change them without breaking the class. Code outside doesn't need to know the business rules.

You can add public member functions as gatekeepers e.g GetBalance() would return an account balance,

never assume there is a GetSomething() for every variable.

you don't always need a SetSomething().

Add as few public member functions as you can. use private functions to keep from repeating code.

The more that is encapsulated the better. Changes in one part of the code doesn't affect other places

Easier for the developer and less likely to cause bugs elsewhere.
